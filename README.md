#👉 What is JVM? Explain its role in Java architecture.
## Ans : JVM (Java Virtual Machine) is an abstract machine that provides runtime environment to execute Java bytecode.
It loads the .class file, verifies the bytecode, and executes it using the execution engine.
JVM is platform dependent because each operating system has its own JVM implementation, but the bytecode is platform independent, which makes Java “Write Once, Run Anywhere”.


#2. Explain complete Java execution flow.
## Ans: First, the developer writes source code in a .java file.
This file is compiled by the Java compiler (javac), which is part of JDK.
The compiler converts the source code into bytecode and generates a .class file.
This bytecode is platform independent.
Then JRE provides the runtime environment, and JVM loads the .class file using the ClassLoader.
The bytecode is verified by the Bytecode Verifier for security.
After that, the Execution Engine converts bytecode into machine code using the Interpreter and JIT compiler.
Finally, the machine code is executed by the system, and we see the output.


#3 👉 What is the difference between JDK, JRE, and JVM?
## Ans: JVM (Java Virtual Machine) is responsible for executing the bytecode. It converts bytecode into machine code and runs the program. JVM is platform dependent because each OS has its own JVM implementation.

JRE (Java Runtime Environment) provides the environment to run Java applications. It contains JVM and core libraries  like java.lang, java.util required for execution.

JDK (Java Development Kit) is used for development. It contains JRE plus development tools like the Java compiler (javac), debugger, and other utilities.


#4👉 What is bytecode?
Why is it called platform independent?
And how does JVM make Java “Write Once, Run Anywhere”?
## Ans: Bytecode is the intermediate code generated by the Java compiler (javac) after compiling the .java file. It is stored in the .class file.
Bytecode is platform independent because it is not machine-specific code.
It is designed to run on any system that has a compatible JVM.
The JVM on each operating system converts the same bytecode into platform-specific machine code, which makes Java follow the principle of “Write Once, Run Anywhere.”


#5: What are the main components of JVM?
##Ans: JVM mainly consists of three major components:

1️⃣ ClassLoader Subsystem
It loads the .class file into memory.
It also verifies the bytecode using the Bytecode Verifier for security.

2️⃣ Runtime Data Areas (Memory Areas)

Heap → stores objects

Stack → stores method calls and local variables

Method Area → stores class metadata, static variables

PC Register → stores address of current instruction

Native Method Stack → used for native methods

3️⃣ Execution Engine

Interpreter → executes bytecode line by line

JIT Compiler → converts frequently used bytecode into native machine code for faster execution

Together, these components allow JVM to execute Java programs efficiently.



#6. Diff between Heap and Stack memory in jvm?
## Ans: Heap memory is used to store objects and instance variables. It is shared among all threads, and garbage collection works on heap memory.

Stack memory is used to store method calls, local variables, and references. Each thread has its own stack memory.

If heap memory is full, it throws OutOfMemoryError.
If stack memory exceeds its limit due to deep recursion, it throws StackOverflowError.


# 7. 👉 What is the difference between Heap and Stack memory in JVM?
## ans: Objects are stored in heap memory because heap supports dynamic memory allocation and object sharing across methods and threads. 
Stack memory is limited to method calls and local variables and gets destroyed when the method execution completes, so it cannot store objects safely.



#8. What is Garbage Collection ?
Garbage Collection is a mechanism in Java that automatically removes unused objects from heap memory.
It helps prevent memory leaks and manages memory efficiently.
An object becomes eligible for garbage collection when there are no active references pointing to it.
The JVM automatically performs garbage collection.

#Types of variable?
There are 3 types of variables in Java:

1️⃣ Local Variable

Declared inside method

Stored in stack

2️⃣ Instance Variable

Declared inside class but outside method

Belongs to object

Stored in heap

3️⃣ Static Variable

Declared with static keyword

Shared among all objects

Stored in method area


#diff between static and instance variable?
nstance variables are declared inside a class but outside methods. Each object has its own copy of instance variables and they are accessed using object reference.

Static variables are declared using the static keyword. Only one copy of static variable exists and it is shared among all objects. It can be accessed using class name.


#where staic , local and instance variable are stored?
Instance variables are stored in heap memory because they are part of the object.
Static variables are stored in method area because they belong to the class.
Local variables are stored in stack memory.


#Referce Data type
Non-primitive data types are reference types in Java that store memory addresses of objects rather than actual values. Examples include String, Arrays, Classes, and Interfaces. They can call methods and their default value is null.

# what is String pool?
In Java, the String Pool is a special area in the heap memory where string literals are stored.
It helps reuse strings, saving memory and improving performance.
When you create a string literal, Java checks the pool first — if the string exists, it reuses the object instead of creating a new one.
